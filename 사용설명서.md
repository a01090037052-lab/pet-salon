## 1. 소개

**펫살롱**은 애견 미용 샵을 위한 고객 관리 프로그램입니다.

| 할 수 있는 것 | 앱의 특징 |
|--------------|----------|
| 고객 + 반려견 정보 관리 | 완전 무료, 구독료 없음 |
| 예약 관리 (월간 캘린더 + 시간 충돌 감지) | 오프라인 사용 가능 |
| 미용 기록 + 매출 추적 | 데이터가 내 기기에 저장 |
| 반려견 사진 관리 | GitHub Pages로 어디서든 접속 |
| 오늘/주간/월간 매출 시각화 | iOS / Android / PC 지원 |
| 월별 매출 아카이브 | JSON 파일로 백업/복원 |
| 재방문 고객 알림 | 예약 알림 (브라우저 알림) |
| 미용사별 매출 분석 | 외부 라이브러리 의존성 제로 |
| 예약 시 미용 기록 자동 등록 | 홈 화면 추가로 앱처럼 사용 |
| 고객/미용 검색형 선택 | 순수 HTML/JS로 영구 사용 가능 |

## 2. 사용 방식

### 메인: GitHub Pages (권장)

```
앱 주소: https://a01090037052-lab.github.io/pet-salon/
```

- **모바일 메인** + PC 서브 사용에 최적
- 어디서든 URL로 접속
- 데이터는 각 기기 브라우저에 저장 (IndexedDB)
- 완전 무료, 영구 사용 가능
- 업그레이드는 git push만 하면 자동 배포

### 서브: PC 서버 모드 (선택)

- PC에서 `node server.js` 실행
- 같은 WiFi에서 PC + 모바일 데이터 공유
- 데이터는 PC의 `data.json`에 저장

| | GitHub Pages (메인) | PC 서버 (서브) |
|---|---|---|
| **접속 방법** | URL 어디서든 | 같은 WiFi에서만 |
| **데이터 저장** | 기기별 브라우저 | PC의 data.json |
| **기기 간 공유** | 수동 (백업/복원) | 자동 |
| **오프라인** | 가능 (PWA) | PC 서버 필요 |
| **추천 상황** | 모바일 메인 사용 | 매장 PC + 모바일 함께 |

| iPhone (Safari) | Android (Chrome) |
|-----------------|------------------|
| 1. Safari에서 앱 주소 접속 | 1. Chrome에서 앱 주소 접속 |
| 2. 하단 **공유** 버튼 탭 | 2. 우측 상단 **메뉴**(⋮) 탭 |

## 4. PC 서버 모드 (선택)

### 준비물

| 항목 | 설명 |
|------|------|
| Node.js | https://nodejs.org 에서 LTS 버전 설치 |
| 같은 WiFi | PC와 모바일이 같은 네트워크 |

### 실행

```
Mac     →  start.command 더블클릭
Windows →  start.bat 더블클릭
```

접속:
- PC → http://localhost:3000
- 모바일 → 터미널에 표시된 주소 (예: http://192.168.0.74:3000)

## 5. 초기 설정 (처음 사용 시)

### 1단계: 매장 정보 입력

**설정** 메뉴 > **매장 정보** 카드

| 항목 | 예시 |
|------|------|
| 매장 이름 | 해피펫살롱 |
| 연락처 | 02-1234-5678 |
| 주소 | 서울시 강남구 ... |
| 재방문 알림 기준 | 30일 (기본값) |

### 2단계: 알림 설정

**설정** 메뉴 > **예약 알림** 카드

1. **알림 활성화** 토글 켜기
2. **알림 시간** 선택 (예: 30분 전)
3. 브라우저 알림 권한 허용

### 3단계: 미용사 등록

**설정** > **미용사 관리**에서 미용사 이름 추가 후 저장

### 4단계: 서비스 메뉴 등록

**서비스 메뉴** > **기본 서비스 자동 등록** 클릭 → 10개 기본 서비스 자동 생성

| 서비스 | 소형 | 중형 | 대형 |
|--------|------|------|------|
| 전체 미용 | 50,000 | 60,000 | 80,000 |
| 목욕 | 30,000 | 40,000 | 50,000 |
| 위생 미용 | 15,000 | 20,000 | 25,000 |
| 가위 커트 | 55,000 | 65,000 | 85,000 |
| ... 외 6개 | | | |

### 5단계: 첫 고객 + 반려견 등록

1. **고객 관리** > **+ 새 고객**
2. 이름, 연락처 입력 후 저장
3. 고객 상세에서 **+ 반려견 등록**
4. 반려견 정보 + **사진 업로드** + 성격/건강/알러지 입력

## 6. 일일 사용 가이드

### 아침 - 하루 시작

```
대시보드 확인
├── 백업 알림이 있으면 → "백업하기" 클릭
├── 오늘 예약 목록 확인
├── 이번 주 매출 차트 확인
├── 재방문 알림 → 연락할 고객 확인
└── 이번 주 예약 미리 확인
```

### 전화 예약 받을 때

1. **예약 관리** > **+ 새 예약**
2. 고객 이름 또는 전화번호로 **검색** (검색형 선택)
3. 반려견 선택 (고객 선택 시 자동 표시)
4. 날짜 + 시간 + 미용사 + 서비스 선택
5. 필요 시 **"미용 기록도 함께 등록"** 체크
6. **저장**

> 예약 관리에서 **캘린더 보기** 버튼을 누르면 한 달 예약을 한눈에 확인 가능

### 미용 완료 후

**방법 A** (빠른 방법): 대시보드 또는 예약 목록에서 **기록** 버튼 클릭
**방법 B**: 미용 기록 > **+ 새 기록** 클릭

| 자동 기능 | 설명 |
|----------|------|
| 고객 검색 선택 | 이름/전화번호로 즉시 검색 |
| 반려견 사이즈 감지 | 몸무게 기반 소형/중형/대형 자동 설정 |
| 가격 자동 계산 | 서비스 체크 시 사이즈에 맞는 가격 합산 |
| 특이사항 표시 | 반려견 성격/건강/알러지 자동 표시 |
| 예약 정보 연동 | 예약에서 "기록" 클릭 시 전부 자동 입력 |

### 마감 - 매출 확인

미용 기록 페이지에서:
- **오늘 매출** 카드 클릭 → 오늘 상세 내역
- **이번 주 매출** 카드 클릭 → 일별 바 차트
- **이번 달 매출** 카드 클릭 → 일별 바 차트
- 하단 **월별 아카이브**에서 과거 매출 확인

## 7. 메뉴별 상세 기능

### 대시보드 (홈)

| 영역 | 내용 |
|------|------|
| 인사말 배너 | 시간대별 인사 + 매장명 + 오늘 요약 |
| 백업 알림 | 7일 이상 백업 안 했으면 경고 |
| 통계 카드 (4개) | 오늘 예약 / 이번 달 매출 / 총 고객 / 오늘 매출 |
| **이번 주 매출 차트** | 월~일 일별 막대 그래프, 오늘 초록색 강조 |
| 6개월 매출 차트 | 월별 막대 그래프 |
| 미용사별 매출 | 이번 달 비율 + 금액 |
| 빠른 액션 | 새 고객/반려견/예약/기록 바로가기 |
| 오늘 예약 | 시간순 목록 + 기록 버튼 |
| 재방문 알림 | 기준일 초과 고객 + 예약 버튼 |
| 이번 주 예약 | 향후 7일 예약 |

### 고객 관리

- **검색**: 이름/전화번호 즉시 검색
- **중복 방지**: 같은 전화번호 등록 시 경고
- **상세 페이지**: 고객 정보 + 반려견 목록 + 미용 기록 + 바로 예약

### 반려견 관리

- **사진 관리**: 반려견 사진 업로드/변경/삭제
- **상세 정보**: 견종, 몸무게, 생년월일(나이 자동계산), 성별, 중성화, 사이즈
- **특이사항**: 성격/행동, 건강, 알러지, 메모
- **미용 이력**: 전체 미용 기록 조회
- 목록에서 사진 표시 (사진 없으면 강아지 이모지)

### 예약 관리

| 기능 | 설명 |
|------|------|
| **캘린더 보기** | 월간 캘린더 그리드, 날짜별 예약 건수 색상 표시 (0=회색, 1~2=파랑, 3+=빨강), 이전/다음 달 이동 |
| 빠른 필터 | 전체/오늘/내일/이번주 |
| 상태 필터 | 대기/확정/완료/취소/노쇼 |
| **검색형 고객 선택** | 이름/전화번호로 실시간 검색 |
| **미용 기록 자동 등록** | "미용 기록도 함께 등록" 체크 시 예약 저장 후 미용 기록 폼 자동 열림 |
| 시간 충돌 감지 | 같은 시간 같은 미용사/반려견 중복 경고 |
| 상태 바로 변경 | 테이블에서 드롭다운으로 즉시 변경 |

**예약 상태**

| 상태 | 의미 |
|------|------|
| 대기 | 아직 확정 안 됨 |
| 확정 | 고객 확정 |
| 완료 | 미용 완료 |
| 취소 | 고객 취소 |
| 노쇼 | 예약 시간에 안 옴 |

### 미용 기록

| 기능 | 설명 |
|------|------|
| **매출 시각화** | 오늘/이번 주/이번 달 매출 카드 클릭 → 상세 차트 모달 |
| **검색형 고객 선택** | 이름/전화번호 실시간 검색 |
| **월별 아카이브** | 전체 월별 매출 요약, 펼쳐서 상세 내역 확인 |
| 가격 자동 계산 | 서비스 + 사이즈 기반 |
| 특이사항 자동 표시 | 반려견 선택 시 성격/건강/알러지 표시 |
| 결제 수단 | 현금/카드/이체 |

### 서비스 메뉴

- 소형/중형/대형 3단계 가격
- 활성/비활성 토글
- 기본 서비스 10개 자동 등록

### 설정

| 섹션 | 내용 |
|------|------|
| 매장 정보 | 매장명, 연락처, 주소, 재방문 알림 기준일 |
| **예약 알림** | 활성화 토글 + 알림 시간 설정 (10분/30분/1시간/2시간 전) |
| 미용사 관리 | 추가/수정/삭제 |
| 데이터 현황 | 각 항목 건수 |
| 백업 & 복원 | JSON 내보내기/가져오기 |
| 위험 영역 | 전체 초기화 (2중 확인) |

## 9. 데이터 관리 & 백업

### 백업 방법

1. **설정** > **데이터 백업하기** 클릭
2. JSON 파일 다운로드
3. iCloud / Google Drive / 카카오톡에 보관

| 백업 주기 | 권장도 |
|----------|--------|
| 매일 | 가장 안전 |
| 주 2회 | 권장 |
| 주 1회 | 최소 |

> 7일 이상 백업하지 않으면 대시보드에 경고 표시

### 복원 방법

1. **설정** > **데이터 복원하기** 클릭
2. 백업 JSON 파일 선택
3. 확인 (현재 데이터가 백업 내용으로 교체됨)

### 폰 교체 시 데이터 이전

```
현재 폰:  설정 → "데이터 백업하기" → JSON 파일 저장
              ↓
         카카오톡, 에어드롭, 이메일로 전송
              ↓
새 폰:    같은 주소 접속 → 설정 → "데이터 복원하기" → 파일 선택
```

### PC에서 데이터 확인하고 싶을 때

```
모바일:  설정 → 백업 → JSON 파일 저장
            ↓
PC:      같은 URL 접속 → 설정 → 복원 → JSON 파일 선택
```

### 코드 수정 후 배포 (PC에서)

```bash

# 2. 코드 수정 (직접 또는 Claude에게 요청)

# 4. 약 20~30초 후 자동 배포 완료
```

### 모바일에서 업데이트 적용

- 브라우저: **새로고침** (아래로 당기기)
- PWA(홈 화면 앱): 앱 종료 후 다시 열기

### 핵심: 업그레이드해도 데이터는 안전

```
코드(HTML/CSS/JS) = GitHub에서 새로 받음 → 업데이트됨
데이터(고객/예약)  = 기기 브라우저에 저장 → 그대로 유지
```

### 업그레이드 실패 시 복구

```bash
# 최근 커밋 확인
git log --oneline -5

# 문제 있는 커밋 되돌리기
git revert HEAD
git push
```

## 13. 문제 해결

### 데이터가 사라졌어요

1. 백업 파일이 있으면 → **설정 > 데이터 복원하기**
2. 없으면 수동 재입력 필요 (**정기 백업의 중요성!**)

### 업그레이드 후 화면이 이상해요

- 브라우저 캐시 문제. **강제 새로고침** (Ctrl+Shift+R 또는 캐시 삭제 후 접속)
- 그래도 안 되면 `git revert HEAD && git push`로 이전 버전 복구

### 알림이 안 와요

| 확인 | 방법 |
|------|------|
| 알림 활성화 | 설정 > 예약 알림 > 토글 켜기 |
| 브라우저 권한 | 브라우저 설정에서 알림 허용 확인 |
| 앱 열려있어야 함 | 브라우저/PWA가 열려있을 때만 작동 |

### 사진이 업로드 안 돼요

- 파일 크기가 너무 크면 느릴 수 있음
- 지원 형식: JPG, PNG, GIF, WebP

### PC 서버 접속 안 돼요 (서버 모드)

```
체크리스트:
☐ PC와 모바일이 같은 WiFi인가?
☐ 서버가 실행 중인가? (터미널 확인)
☐ 주소 앞에 http:// 를 붙였는가?
☐ IP 주소가 정확한가?
```

## 15. 자주 묻는 질문

**Q. 처음 설정할 때 꼭 해야 하는 건?**
> 매장 정보 입력 → 알림 설정 → 미용사 등록 → 서비스 메뉴 등록

**Q. 홈 화면에 추가는 꼭 해야 하나요?**
> 네. 앱처럼 사용 가능하고, Safari 데이터 자동 삭제도 방지됩니다.

**Q. PC와 모바일 데이터가 달라요.**
> GitHub Pages 모드에서는 각 기기가 독립 데이터입니다. 백업/복원으로 동기화하세요.

**Q. 실수로 고객을 삭제했어요.**
> 백업 파일이 있으면 **설정 > 데이터 복원**으로 복구 가능.

**Q. 여러 매장을 따로 관리하고 싶어요.**
> 같은 URL을 사용해도 기기별로 데이터가 독립적입니다.

**Q. 가격이 자동 계산되지 않아요.**
> 서비스에 가격 설정 + 반려견 사이즈(또는 몸무게) 입력 확인.

**Q. 알림이 앱을 닫아도 오나요?**
> 아니요. 브라우저/PWA가 열려있을 때만 작동합니다.

**Q. 다른 사람도 이 앱을 쓸 수 있나요?**
> 네. URL만 공유하면 됩니다. 데이터는 각자 기기에 별도 저장됩니다.

**Q. 업그레이드하면 데이터가 날아가나요?**
> 아니요. 코드만 업데이트되고 데이터(IndexedDB)는 그대로 유지됩니다.

**Q. GitHub Pages가 없어지면?**
> 앱은 순수 HTML/JS 파일이므로 다른 호스팅으로 파일만 옮기면 됩니다.

**Q. 30년 후에도 사용 가능한가요?**
> HTML/JS는 웹 표준이므로 브라우저가 존재하는 한 동작합니다. 호스팅만 옮기면 됩니다.
