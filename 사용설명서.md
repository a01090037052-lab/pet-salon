## 1. 소개

**펫살롱**은 애견 미용 샵을 위한 고객 관리 프로그램입니다. v3.0에서는 대시보드 강화, 마케팅 기능, 사진 카드, 미용사별 타임테이블 등 대폭 기능이 추가되었습니다.

| 할 수 있는 것 | 앱의 특징 |
|--------------|----------|
| 고객 + 반려견 정보 관리 + 태깅 | 완전 무료, 구독료 없음 |
| 예약 관리 (월간/타임테이블 + 충돌 감지) | 오프라인 사용 가능 |
| 미용 기록 + 실시간 매출 추적 | 데이터가 내 기기에 저장 |
| 반려견 사진 + 풀스크린 라이트박스 | GitHub Pages로 어디서든 접속 |
| 일별/주간/월간/6개월 매출 시각화 | iOS / Android / PC 지원 |
| 미용사별 + 서비스별 매출 분석 | JSON 파일로 백업/복원 |
| 재방문/생일/미수금 알림 (자동 문자) | 예약 알림 (브라우저 알림) |
| 포인트/스탬프 + 프로모션 관리 | 사진 카드 생성 & 공유 (8종 레이아웃) |
| 세무 자료 CSV 내보내기 | 저장소 용량 모니터링 |
| 순수 HTML/JS로 영구 사용 가능 | 전역 검색 (/ 키 또는 돋보기) |

## 2. 사용 방식

### 메인: GitHub Pages (권장)

```
앱 주소: https://a01090037052-lab.github.io/pet-salon/
```

- **모바일 메인** + PC 서브 사용에 최적
- 어디서든 URL로 접속
- 데이터는 각 기기 브라우저에 저장 (IndexedDB)
- 완전 무료, 영구 사용 가능
- 업그레이드는 git push만 하면 자동 배포

### 서브: PC 서버 모드 (선택)

- PC에서 `node server.js` 실행
- 같은 WiFi에서 PC + 모바일 데이터 공유
- 데이터는 PC의 `data.json`에 저장

| | GitHub Pages (메인) | PC 서버 (서브) |
|---|---|---|
| **접속 방법** | URL 어디서든 | 같은 WiFi에서만 |
| **데이터 저장** | 기기별 브라우저 | PC의 data.json |
| **기기 간 공유** | 수동 (백업/복원) | 자동 |
| **오프라인** | 가능 (PWA) | PC 서버 필요 |
| **추천 상황** | 모바일 메인 사용 | 매장 PC + 모바일 함께 |

2. 바로 사용 가능 — **이게 끝입니다!**

### 홈 화면에 추가 (앱처럼 사용) — 필수 권장

| iPhone (Safari) | Android (Chrome) |
|-----------------|------------------|
| 1. Safari에서 앱 주소 접속 | 1. Chrome에서 앱 주소 접속 |
| 2. 하단 **공유** 버튼 탭 | 2. 우측 상단 **메뉴**(⋮) 탭 |
| 3. **"홈 화면에 추가"** | 3. **"홈 화면에 추가"** |
| 4. **"추가"** 탭 | 4. **"추가"** 탭 |

> **중요**: 홈 화면에 추가하면 앱처럼 전체 화면으로 동작하고, Safari의 7일 데이터 삭제 규칙도 적용되지 않아 데이터가 더 안전합니다.

## 4. PC 서버 모드 (선택)

### 준비물

| 항목 | 설명 |
|------|------|
| Node.js | https://nodejs.org 에서 LTS 버전 설치 |
| 같은 WiFi | PC와 모바일이 같은 네트워크 |

### 실행

```
Mac     →  start.command 더블클릭
Windows →  start.bat 더블클릭
```

접속:
- PC → http://localhost:3000
- 모바일 → 터미널에 표시된 주소 (예: http://192.168.0.74:3000)

## 5. 초기 설정 (처음 사용 시)

### 1단계: 매장 정보 입력

**설정** 메뉴 > **매장 관리** 탭 > **매장 정보** 카드

| 항목 | 예시 |
|------|------|
| 매장 이름 | 해피펫살롱 |
| 연락처 | 02-1234-5678 |
| 주소 | 서울시 강남구 ... |
| 재방문 알림 기준 | 30일 (기본값) |

### 2단계: 알림 설정

**설정** 메뉴 > **알림·메시지** 탭

1. **예약 알림 활성화** 토글 켜기
2. **알림 시간** 선택 (예: 30분 전)
3. **메시지 템플릿** 4종 확인/수정:
   - 재방문 알림 문자
   - 예약 확인 문자
   - 생일 축하 문자
   - 미용 완료 문자
4. 브라우저 알림 권한 허용

### 3단계: 미용사 등록

**설정** > **매장 관리** 탭 > **미용사 관리**에서 미용사 이름 추가 후 저장

### 4단계: 휴무일 설정

**설정** > **매장 관리** 탭 > **휴무일 설정**에서 정기 휴무일 표시

### 5단계: 서비스 메뉴 등록

**서비스 메뉴** > **기본 서비스 자동 등록** 클릭 → 10개 기본 서비스 자동 생성

| 서비스 | 소형 | 중형 | 대형 |
|--------|------|------|------|
| 전체 미용 | 50,000 | 60,000 | 80,000 |
| 목욕 | 30,000 | 40,000 | 50,000 |
| 위생 미용 | 15,000 | 20,000 | 25,000 |
| 가위 커트 | 55,000 | 65,000 | 85,000 |
| ... 외 6개 | | | |

- 각 서비스의 3단계 가격 설정 (소형/중형/대형)
- 활성/비활성 토글로 사용 여부 관리
- 컴팩트 칩 선택으로 자주 쓰는 서비스 표시 가능

### 6단계: 마케팅 설정

**설정** > **마케팅** 탭

- **포인트 설정**: 서비스별 적립률 또는 고정액
- **스탬프 설정**: 회차, 보상 규정
- **프로모션 관리**: 활성 프로모션 추가 (할인율/할인액, 기간)

### 7단계: 사진 카드 커스터마이징

**설정** > **사진 카드** 탭

- **레이아웃** 8종 중 선택 (미용 전후, 정면, 옆모습 등)
- **시즌 테마** 7종 (봄/여름/가을/겨울/크리스마스/설날/등)
- **글꼴/테두리/스티커** 커스터마이징
- **매장 로고** 업로드
- **배경 이미지** 추가

### 8단계: 첫 고객 + 반려견 등록

1. **고객 관리** > **+ 새 고객**
2. 이름, 연락처, 생일 입력 후 저장
3. 고객 상세에서 **+ 반려견 등록**
4. 반려견 정보 + **사진 업로드** + 권장 미용 주기 + 성격/건강/알러지 입력

## 6. 일일 사용 가이드

### 아침 - 하루 시작

```
대시보드 확인
├── 백업 알림이 있으면 → "백업하기" 클릭
├── 오늘 예약 목록 확인 (상태별 통계 카드)
├── 오늘 예약 상세 리스트 + 기록 버튼
├── 이번 주 매출 차트 확인
├── 재방문 알림 → 연락할 고객 확인 + 문자/전화 버튼
├── 생일 알림 → 축하 문자 버튼
├── 미수금 현황 → 결제 처리 버튼
├── 관리 필요 고객 (불만족 이력) 확인
├── 활성 프로모션 배너 확인
└── 저장소 용량 경고 확인
```

### 전화 예약 받을 때

1. **예약 관리** > **+ 새 예약**
2. 고객 이름 또는 전화번호로 **검색** (검색형 선택)
3. 반려견 선택 (고객 선택 시 자동 표시)
4. 날짜 + 시간 + 미용사 + 서비스 선택
5. 예상 소요시간 입력 (30분~3시간)
6. 필요 시 **"미용 기록도 함께 등록"** 체크
7. 노쇼 이력이 있으면 경고 표시 확인
8. **저장**

> 예약 관리에서 **캘린더 보기** 또는 **타임테이블 보기** 버튼으로 한눈에 확인 가능. 시간 범위 충돌 자동 감지.

### 미용 완료 후

**방법 A** (빠른 방법): 대시보드 또는 예약 목록에서 **기록** 버튼 클릭
**방법 B**: 미용 기록 > **+ 새 기록** 클릭

| 자동 기능 | 설명 |
|----------|------|
| 고객 검색 선택 | 이름/전화번호로 즉시 검색 |
| 반려견 사이즈 감지 | 몸무게 기반 소형/중형/대형 자동 설정 |
| 가격 자동 계산 | 서비스 체크 시 사이즈에 맞는 가격 합산 |
| 특이사항 표시 | 반려견 성격/건강/알러지 자동 표시 |
| 예약 정보 연동 | 예약에서 "기록" 클릭 시 전부 자동 입력 |
| 프로모션 자동 적용 | 활성 프로모션 자동 적용 (할인/추가요금) |
| 포인트/스탬프 관리 | 사용/적립 입력 |
| 미수금 관리 | 외상 등록 |

### 미용 후 추가 기능

- **미용 전/후 사진 업로드** (자동 압축)
- **고객 만족도** 입력 (만족/보통/불만)
- **최종 금액 자동 계산** (가격 + 할인/추가요금)
- **다음 예약 추천** (미용 완료 후 자동 표시)
- **미용 완료 문자 자동 발송** 프롬프트

### 마감 - 매출 확인

미용 기록 페이지에서:
- **오늘 매출** 카드 클릭 → 오늘 상세 내역
- **이번 주 매출** 카드 클릭 → 일별 바 차트 (월~일)
- **이번 달 매출** 카드 클릭 → 일별 바 차트
- **6개월 매출** 차트 확인 (추이 분석)
- **미용사별 매출** 확인 (이번 달 비율 + 금액)
- **서비스별 매출** 확인 (인기 서비스 분석)
- 하단 **월별 아카이브**에서 과거 매출 확인
- **세무 자료 CSV 내보내기** (월 단위)
- **일일 정산표** 출력/확인

## 7. 메뉴별 상세 기능

### 대시보드 (홈)

| 영역 | 내용 |
|------|------|
| **인사말 배너** | 시간대별 인사 + 매장명 + 오늘 요약 |
| **백업 알림** | 7일 이상 백업 안 했으면 경고 |
| **저장소 용량 경고** | 저장소 부족 시 경고 (사진 압축 권장) |
| **통계 카드 (4개)** | 오늘 예약/이번 달 매출/총 고객수/오늘 매출 |
| **이번 주 매출 차트** | 월~일 일별 막대 그래프, 오늘 초록색 강조 |
| **6개월 매출 차트** | 월별 막대 그래프 |
| **미용사별 매출** | 이번 달 비율 + 금액 표시 |
| **서비스별 매출** | 인기 서비스 분석 |
| **빠른 액션** | 새 고객/반려견/예약/기록 바로가기 |
| **오늘 예약 목록** | 시간순 목록 + 기록 버튼 + 상태 표시 |
| **재방문 알림** | 기준일 초과 고객 + 문자/전화 버튼 + 반려견별 개별 주기 지원 |
| **생일 알림** | 고객/반려견 생일 다가오면 축하 문자 버튼 |
| **미수금 현황** | 외상 잔액 표시 + 결제 처리 버튼 |
| **관리 필요 고객** | 불만족 이력 있는 고객 |
| **활성 프로모션 배너** | 진행 중인 프로모션 표시 |
| **이번 주 예약** | 향후 7일 예약 목록 |

### 고객 관리

**목록 기능**:
- **검색**: 이름/전화번호 즉시 검색
- **정렬**: 이름/방문일/등록일/방문횟수 정렬
- **태깅**: VIP/신규/단골/주의 태그 추가 (필터 지원)
- **장기 미방문 경고**: 30일+/60일+ 배지 표시
- **노쇼 이력 표시**
- **모바일 카드 뷰**: 카드 형태로 정보 표시

**고객 상세 페이지**:
- **기본 정보**: 이름, 연락처, 주소, 생일
- **방문 통계**: 총 방문 횟수, 총 지출, 평균 객단가, 평균 방문 주기
- **미수금 잔액**: 현재 외상액
- **포인트/스탬프 잔액**: 적립 포인트 + 스탬프 수
- **반려견 목록**: 고객의 모든 반려견 표시
- **미용 기록**: 고객의 전체 미용 기록 조회
- **빠른 액션**: 문자/전화 링크 + 바로 예약

### 반려견 관리

**등록/수정**:
- **사진 관리**: 자동 압축 지원, 업로드/변경/삭제 가능
- **기본 정보**: 반려견 이름, 견종, 생년월일(나이 자동계산), 성별, 중성화 여부, 몸무게
- **사이즈**: 소형/중형/대형 자동 판정 또는 수동 설정
- **선호 미용 스타일**: 자유 메모 필드
- **권장 미용 주기**: 개별 주기 설정 (재방문 알림 기준)
- **특이사항**: 성격/행동, 건강, 알러지, 일반 메모

**목록**:

### 다른 매장에 공유하기 전에 알아두세요

**데이터는 절대 섞이지 않습니다!**

- A 매장의 고객 정보가 B 매장에 보이지 않음
- 각 매장이 독립적으로 자신의 데이터만 관리 가능

### 상대방에게 공유하는 방법

1. **앱 URL 공유**: `https://a01090037052-lab.github.io/pet-salon/`
2. **상대방이 접속하면 바로 사용 가능**
3. **각자 초기 설정만 하면 끝**

### 상대방의 첫 사용 가이드

상대방에게 다음 순서대로 안내하세요:

#### Step 1: 앱 접속 (2분)
1. 브라우저에서 URL 접속
2. 홈 화면에 추가 (iPhone: 공유 > 홈 화면에 추가 / Android: 메뉴 > 홈 화면에 추가)
3. 홈 화면의 앱 아이콘 탭

#### Step 2: 필수 초기 설정 (10분)
**더보기 > 설정** 이동

1. **매장 관리 탭**:
   - 매장 이름 입력 (예: "행복한 미용실")
   - 연락처, 주소 입력
   - 미용사 이름 3명 정도 등록 (나중에 수정 가능)

2. **알림·메시지 탭**:
   - "예약 알림" 토글 ON
   - 알림 시간 "30분 전" 선택 (기본값 권장)

3. **마케팅 탭** (선택):
   - 포인트/스탬프 사용하면 설정, 안 쓰면 패스

#### Step 3: 기본 서비스 등록 (1분)
**더보기 > 서비스 메뉴** > **"기본 서비스 자동 등록"** 클릭
- 기본 10개 서비스 + 기본 가격 자동 생성
- 나중에 가격 수정/추가 가능

#### Step 4: 첫 데이터 입력 (5분)
**고객** 탭 > **+ 새 고객**
1. 이름, 연락처 입력 후 저장
2. 저장된 고객 탭 > **+ 반려견 추가**
3. 반려견 이름, 몸무게 입력 + 사진 업로드
4. 저장

#### Step 5: 첫 예약/기록 (5분)
**예약** 탭 > **+ 새 예약**
1. 위에서 등록한 고객 검색 선택
2. 날짜, 시간, 미용사, 서비스 선택
3. "미용 기록도 함께 등록" 체크 (선택)
4. 저장

**미용 기록** 탭에서 "기록" 또는 "+ 새 기록"으로 매출 기록

## 10. 데이터 관리 & 백업

### 백업 방법

1. **설정** > **데이터** 탭 > **데이터 백업하기** 클릭
2. JSON 파일 다운로드
3. iCloud / Google Drive / 카카오톡에 보관

| 백업 주기 | 권장도 |
|----------|--------|
| 매일 | 가장 안전 |
| 주 2회 | 권장 |
| 주 1회 | 최소 |

> 7일 이상 백업하지 않으면 대시보드에 경고 표시

### 복원 방법

1. **설정** > **데이터** 탭 > **데이터 복원하기** 클릭
2. 백업 JSON 파일 선택
3. 확인 (현재 데이터가 백업 내용으로 교체됨)

### 폰 교체 시 데이터 이전

```
현재 폰:  설정 → "데이터 백업하기" → JSON 파일 저장
             ↓
        카카오톡, 에어드롭, 이메일로 전송
             ↓
새 폰:    같은 주소 접속 → 설정 → "데이터 복원하기" → 파일 선택
```

### PC에서 데이터 확인하고 싶을 때

```
모바일:  설정 → 백업 → JSON 파일 저장
            ↓
PC:      같은 URL 접속 → 설정 → 복원 → JSON 파일 선택
```

| **기기별 데이터 분리** | PC와 모바일 데이터는 별개 |
| **반려견 사진** | 많아지면 백업 파일 커짐. 작게 촬영 권장 |
| **알림** | 앱이 열려있을 때만 작동 (브라우저 알림) |
| **저장소 용량** | 사진이 많으면 압축 권장. 저장소 경고 확인 |

## 14. 문제 해결

### 데이터가 사라졌어요

1. 백업 파일이 있으면 → **설정 > 데이터** 탭 > **데이터 복원하기**
2. 없으면 수동 재입력 필요 (**정기 백업의 중요성!**)

### 업그레이드 후 화면이 이상해요

- 브라우저 캐시 문제. **강제 새로고침** (Ctrl+Shift+R 또는 캐시 삭제 후 접속)
- 그래도 안 되면 `git revert HEAD && git push`로 이전 버전 복구

### 알림이 안 와요

| 확인 | 방법 |
|------|------|
| 알림 활성화 | 설정 > 알림·메시지 탭 > 예약 알림 > 토글 켜기 |
| 브라우저 권한 | 브라우저 설정에서 알림 허용 확인 |
| 앱 열려있어야 함 | 브라우저/PWA가 열려있을 때만 작동 |

### 사진이 업로드 안 돼요

- 파일 크기가 너무 크면 느릴 수 있음
- 지원 형식: JPG, PNG, GIF, WebP
- 저장소 용량 경고 확인 (사진 압축 권장)

### PC 서버 접속 안 돼요 (서버 모드)

```
체크리스트:
☐ PC와 모바일이 같은 WiFi인가?
☐ 서버가 실행 중인가? (터미널 확인)
☐ 주소 앞에 http:// 를 붙였는가?
☐ IP 주소가 정확한가?
```

### 예상 소요시간 설정 후에도 시간 충돌이 감지돼요

- 예상 소요시간은 충돌 감지에 사용됨
- 예: 14:00 시작, 1시간 소요 → 14:00~15:00 점유
- 같은 미용사 또는 반려견이 이 시간에 다른 예약이 있으면 경고

### 포인트/스탬프가 적립이 안 돼요

- 마케팅 탭에서 포인트/스탐프 설정 확인
- 미용 기록에서 "포인트 적립" 또는 "스탬프 적립" 필드 확인

### 프로모션이 자동 적용이 안 돼요

- 마케팅 탭에서 프로모션이 "활성"으로 설정되었는지 확인
- 프로모션 기간 확인 (현재 날짜가 기간 범위 내인가?)
- 프로모션 적용 대상 확인 (전체 또는 특정 고객?)

## 16. 자주 묻는 질문

**Q. 처음 설정할 때 꼭 해야 하는 건?**
> 매장 정보 입력 → 미용사 등록 → 서비스 메뉴 등록 (자동) → 첫 고객 + 반려견 등록. 나머지는 필요할 때 설정.

**Q. 홈 화면에 추가는 꼭 해야 하나요?**
> 네. 앱처럼 사용 가능하고, Safari 데이터 자동 삭제도 방지됩니다.

**Q. PC와 모바일 데이터가 달라요.**
> GitHub Pages 모드에서는 각 기기가 독립 데이터입니다. 백업/복원으로 동기화하세요. PC 서버 모드를 쓰면 자동 동기화됩니다.

**Q. 실수로 고객을 삭제했어요.**
> 백업 파일이 있으면 **설정 > 데이터** 탭 > **데이터 복원**으로 복구 가능.

**Q. 여러 매장을 따로 관리하고 싶어요.**
> 같은 URL을 사용해도 기기별로 데이터가 독립적입니다. 각 기기에서 독립 데이터 관리 가능.

**Q. 가격이 자동 계산되지 않아요.**
> 서비스에 가격 설정 + 반려견 사이즈(또는 몸무게) 입력 확인.

**Q. 알림이 앱을 닫아도 오나요?**
> 아니요. 브라우저/PWA가 열려있을 때만 작동합니다. 푸시 알림이 아닌 브라우저 알림입니다.

**Q. 다른 사람도 이 앱을 쓸 수 있나요?**
> 네. URL만 공유하면 됩니다. 데이터는 각자 기기에 별도 저장됩니다.

**Q. 업그레이드하면 데이터가 날아가나요?**
> 아니요. 코드만 업데이트되고 데이터(IndexedDB)는 그대로 유지됩니다.

**Q. GitHub Pages가 없어지면?**
> 앱은 순수 HTML/JS 파일이므로 다른 호스팅으로 파일만 옮기면 됩니다.

**Q. 30년 후에도 사용 가능한가요?**
> HTML/JS는 웹 표준이므로 브라우저가 존재하는 한 동작합니다. 호스팅만 옮기면 됩니다.

**Q. 포인트와 스탬프의 차이가 뭔가요?**
> 포인트는 금액 기반 (예: 1,000원마다 100포인트), 스탬프는 회차 기반 (예: 10번 방문하면 보상). 둘 다 사용/적립 관리 가능.

**Q. 프로모션은 어떻게 적용되나요?**
> 마케팅 탭에서 프로모션 추가 (할인율/할인액, 기간, 대상) → 미용 기록 등록 시 활성 프로모션 자동 적용. 수동 조정도 가능.

**Q. 미수금(외상)은 어떻게 관리하나요?**
> 미용 기록에서 "미수금액" 입력 → 대시보드 "미수금 현황"에 누적 → "결제 처리" 버튼으로 정산.

**Q. 사진 카드는 어떻게 만들어요?**
> 미용 기록 저장 후 → "사진 카드 생성" → 8종 레이아웃 + 7종 테마 선택 → 이미지 다운로드 → SNS 공유.

**Q. 노쇼 처리는 어떻게 하나요?**
> 예약 상태를 "노쇼"로 변경 → 시스템이 고객 노쇼 이력 기록 → 다음 예약 등록 시 경고 표시.

**Q. 세무 신고 때 필요한 자료는?**
> 설정 > 데이터 탭 > "세무 자료 내보내기" → 월별 CSV (날짜/고객/서비스/금액) 다운로드 → 세무사에게 제출.

**Q. 반려견의 권장 미용 주기 설정은 뭐하는 건가요?**
> 반려견 상세에서 "권장 미용 주기" 입력 (예: 30일) → 대시보드 "재방문 알림"이 이 주기를 기준으로 작동. 반려견마다 다르게 설정 가능.
